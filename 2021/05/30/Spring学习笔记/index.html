发表于 2021-05-30 更新于 2021-06-06 分类于 后端 ， Spring

1. 简介

Spring是Java EE编程领域的一个轻量级开源框架，是为了解决企业级编程开发中的复杂性，实现敏捷开发的应用型框架 。

1.1 Spring优点

  • 控制反转（IoC，Inversion of Control）
  • 面向切面编程（AOP，Aspect Oriented Programming）
  • 支持事务
  • 非侵入式
  • …

1.2 Spring组成

2. 控制反转 IoC

控制反转是一种编程思想。

2.1 对比传统模式

某个Java对象（调用者）需要调用另一个Java对象（被调用者，即被依赖对象）

传统模式：

  • 调用者采用“new 被调用者”方式创建，强耦合

Spring框架：

  • 对象的实例不再由调用者创建，而是由Spring容器创建。控制权由应用代码转移到Spring容器，控制权发生了反转。

2.3 Spring IoC容器创建Bean（依赖注入）

依赖注入：Dependency Injection，是IoC思想的一种实现方式。

  • 依赖
    • bean对象的创建依赖于Spring容器
  • 注入
    • bean对象的属性由Spring容器注入

3.3 import

合并Spring配置文件。

3.4 scope

bean的作用域。

作用域 说明
singleton 在Spring容器中将只有一个实例，无论有多少个Bean引用它，始终将指向同一个对象，默认
prototype 每次通过Spring容器获取的prototype定义的Bean时，容器都将创建一个新的Bean实例
request 在一次HTTP请求中，容器会返回该Bean的同一个实例，且仅在当前HTTP Request内有效
session 在一次HTTP Session中，容器会返回该Bean的同一个实例，且仅在当前HTTP Session内有效
application 为每个ServletContext对象创建一个实例，仅在Web相关的ApplicationContext中生效
websocket 为每个websocket对象创建一个实例，仅在Web相关的ApplicationContext中生效

4.Bean的装配方式

4.1在bean中显式装配

4.2 自动装配

开启自动装配

4.2.1 通过指定bean中的autowire来隐式装配

指定autowire后的bean会自动在上下文寻找对应的其他bean进行属性注入。

autowire：

  • byName
    • 找到id与setName方法名中的set后面的name对应其他bean进行注入
    • 需要保证id上下文唯一
  • byType
    • 找到类型与要装配的bean的属性类型对应的其他bean进行注入
    • 需要保证类型上下文唯一

4.2.2 通过注解实现

注解名：@Autowired

此方法实现自动装配，默认按照类型进匹配对应的bean；如果此类型有多个bean，则按名称进行匹配。

可以通过@Qualifier注解来指定按名称匹配时有资格被匹配的bean。

4.3 基于注解的装配

  • 在需要注册bean的类上加注解@Component

  • 在applicationContext.xml中扫描加了注解的类所在的包，扫描后会对对应的类生成一个bean

6. AOP

6.1 简介

6.1.1 AOP

  • Aspect Oriented Programming，面向切面编程。

6.1.2 优点

  • 每个事物逻辑位于一个位置, 代码不分散, 便于维护和升级。
  • 业务模块更简洁, 只包含核心业务代码。

6.1.3 相关概念

  • 切面(Aspect): 横切关注点(跨越应用程序多个模块的功能)被模块化的特殊对象。
  • 通知(Advice): 切面必须要完成的工作。（切面类中的每一个方法）
  • 目标(Target): 被通知的对象。
  • 代理(Proxy): 向目标对象应用通知之后创建的对象。
  • 连接点（Joinpoint）：程序执行的某个特定位置——物理意义上的概念。
  • 切点（pointcut）：每个类都拥有多个连接点–逻辑意义上的概念。
